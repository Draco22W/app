# 📢 重要说明 - GET请求错误是正常的

## ✅ 这些错误可以忽略

你看到的这些错误信息：
```json
{
  "error": "此端点仅支持POST请求",
  "method": "GET", 
  "url": "/api/proxy"
}
```

**这是完全正常的！** 不会影响功能使用。

## 🤔 为什么会出现这些请求？

### 可能的原因：
1. **浏览器预检请求** - 浏览器自动发送的安全检查
2. **开发工具探测** - 开发者工具的自动请求
3. **网络监控工具** - 某些网络工具的探测请求
4. **浏览器扩展** - 浏览器插件的后台请求

### 为什么返回错误？
- API代理端点 `/api/proxy` 只接受POST请求
- GET请求会被正确拒绝并返回说明信息
- 这是**安全设计**，防止误用
